Introducción

Express.js es un framework web minimalista, flexible y rápido para Node.js. Proporciona un conjunto robusto de características para desarrollar aplicaciones web y APIs. Este apunte te guiará paso a paso para crear proyectos Express desde la terminal de Linux, ya sea utilizando herramientas automatizadas o configurando todo manualmente.

Requisitos previos:

  • Node.js y npm instalados en tu sistema Linux
  • Conocimientos básicos de comandos de terminal
  • Editor de texto (como Visual Studio Code, Sublime Text, Vim, etc.)

Verificar la instalación de Node.js y npm

Antes de comenzar, asegúrate de tener Node.js y npm correctamente instalados:

usuario@linux:~$ node -v
v18.16.0
usuario@linux:~$ npm -v
9.5.1

Si no tienes Node.js instalado, puedes instalarlo en distribuciones basadas en Debian/Ubuntu con:

usuario@linux:~$ sudo apt update
usuario@linux:~$ sudo apt install nodejs npm

O para distribuciones basadas en Red Hat/Fedora:

usuario@linux:~$ sudo dnf install nodejs npm

Consejo: Para obtener versiones más recientes de Node.js, considera usar NVM (Node Version Manager) que permite instalar y gestionar múltiples versiones de Node.js en tu sistema.

Métodos para iniciar un proyecto Express

Existen dos enfoques principales para iniciar un proyecto Express en Linux:

  1. Usar Express Generator: una herramienta oficial que crea rápidamente la estructura base de una aplicación.
  2. Configuración manual: crear la estructura del proyecto desde cero, con mayor control sobre cada componente.

Vamos a explorar ambos métodos en detalle.

Método 1: Usando Express Generator

Express Generator es una herramienta oficial que crea rápidamente un esqueleto de aplicación con estructura de directorios predefinida y configuración básica.

Paso 1: Instalar Express Generator globalmente

usuario@linux:~$ npm install -g express-generator

La bandera -g instala el paquete globalmente en tu sistema, permitiéndote usar el comando express desde cualquier ubicación.

Paso 2: Crear una nueva aplicación Express

usuario@linux:~$ express mi-aplicacion-express

Por defecto, Express Generator utiliza el motor de plantillas Jade/Pug. Si prefieres otro motor, puedes especificarlo con la opción --view:

usuario@linux:~$ express --view=ejs mi-aplicacion-express
# O para usar Handlebars
usuario@linux:~$ express --view=hbs mi-aplicacion-express

También puedes crear una aplicación sin ninguna vista, útil para APIs:

usuario@linux:~$ express --no-view mi-aplicacion-express

Paso 3: Acceder al directorio e instalar dependencias

usuario@linux:~$ cd mi-aplicacion-express
usuario@linux:~/mi-aplicacion-express$ npm install

Paso 4: Iniciar la aplicación

usuario@linux:~/mi-aplicacion-express$ npm start

Tu aplicación estará disponible en http://localhost:3000

Estructura de directorios generada

mi-aplicacion-express/
├── app.js
├── bin/
│   └── www
├── package.json
├── public/
│   ├── images/
│   ├── javascripts/
│   └── stylesheets/
│       └── style.css
├── routes/
│   ├── index.js
│   └── users.js
└── views/
    ├── error.pug
    ├── index.pug
    └── layout.pug

Consejo: Puedes ver todas las opciones disponibles de Express Generator con:

usuario@linux:~$ express --help

Método 2: Configuración manual

La configuración manual te da mayor control sobre la estructura y dependencias de tu proyecto. Es ideal para aprender los conceptos fundamentales de Express o cuando necesitas una configuración personalizada.

Paso 1: Crear un directorio para el proyecto

usuario@linux:~$ mkdir mi-proyecto-express
usuario@linux:~$ cd mi-proyecto-express

Paso 2: Inicializar el proyecto npm

usuario@linux:~/mi-proyecto-express$ npm init -y

La opción -y acepta todos los valores predeterminados. Si quieres personalizar los detalles del proyecto, omite esta bandera:

usuario@linux:~/mi-proyecto-express$ npm init

Paso 3: Instalar Express

usuario@linux:~/mi-proyecto-express$ npm install express

Paso 4: Crear la estructura básica de directorios

usuario@linux:~/mi-proyecto-express$ mkdir routes public views models controllers

Paso 5: Crear el archivo principal

usuario@linux:~/mi-proyecto-express$ touch app.js

Paso 6: Editar el archivo app.js

Usando tu editor de texto preferido (nano, vim, vs code, etc.), añade el siguiente código básico:

const express = require('express');
const path = require('path');

// Inicializar app
const app = express();
const PORT = process.env.PORT || 3000;

// Middleware para analizar el cuerpo de las solicitudes JSON
app.use(express.json());
app.use(express.urlencoded({ extended: false }));

// Configurar archivos estáticos
app.use(express.static(path.join(__dirname, 'public')));

// Ruta de ejemplo
app.get('/', (req, res) => {
  res.send('¡Hola mundo desde Express!');
});

// Iniciar el servidor
app.listen(PORT, () => {
  console.log(`Servidor corriendo en http://localhost:${PORT}`);
});

Paso 7: Modificar el script de inicio en package.json

Edita el archivo package.json para añadir un script de inicio:

{
  "name": "mi-proyecto-express",
  "version": "1.0.0",
  "description": "",
  "main": "app.js",
  "scripts": {
    "start": "node app.js",
    "dev": "nodemon app.js",
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "express": "^4.18.2"
  }
}

Nota: Para usar el script dev, necesitas instalar nodemon:

usuario@linux:~/mi-proyecto-express$ npm install --save-dev nodemon

Paso 8: Iniciar la aplicación

usuario@linux:~/mi-proyecto-express$ npm start

O para desarrollo con reinicio automático:

usuario@linux:~/mi-proyecto-express$ npm run dev

Mejoras y configuración adicional

Configuración de entornos

Es una buena práctica usar variables de entorno para configurar tu aplicación:

usuario@linux:~/mi-proyecto-express$ npm install dotenv
usuario@linux:~/mi-proyecto-express$ touch .env

Edita el archivo .env:

# Variables de entorno
PORT=3000
NODE_ENV=development
# Añade otras variables según necesidad
DB_URI=mongodb://localhost:27017/miapp

Y modifica tu app.js para usar estas variables:

// Cargar variables de entorno
require('dotenv').config();

// Ahora puedes usar process.env.PORT, etc.

Configurar middleware adicional

Instalar middleware comúnmente usado:

usuario@linux:~/mi-proyecto-express$ npm install morgan cors helmet

Edita app.js para incluir estos middleware:

const express = require('express');
const path = require('path');
const logger = require('morgan');
const cors = require('cors');
const helmet = require('helmet');

const app = express();

// Middleware de seguridad
app.use(helmet());

// Habilitar CORS
app.use(cors());

// Logger de solicitudes HTTP
app.use(logger('dev'));

// Resto del código...

Crear un sistema de rutas modular

Crea archivos de rutas separados en el directorio routes:

usuario@linux:~/mi-proyecto-express$ touch routes/index.js routes/users.js

Edita routes/index.js:

const express = require('express');
const router = express.Router();

router.get('/', (req, res) => {
  res.send('Página de inicio');
});

module.exports = router;

Edita routes/users.js:

const express = require('express');
const router = express.Router();

router.get('/', (req, res) => {
  res.send('Lista de usuarios');
});

router.get('/:id', (req, res) => {
  res.send(`Usuario con ID: ${req.params.id}`);
});

module.exports = router;

Y modifica app.js para usar estas rutas:

// ...
const indexRouter = require('./routes/index');
const usersRouter = require('./routes/users');

// ...

// Usar rutas
app.use('/', indexRouter);
app.use('/users', usersRouter);

Crear un archivo .gitignore

usuario@linux:~/mi-proyecto-express$ touch .gitignore

Añade lo siguiente al archivo .gitignore:

# Dependencias
node_modules/
npm-debug.log*
yarn-debug.log*
yarn-error.log*

# Variables de entorno
.env
.env.local
.env.development.local
.env.test.local
.env.production.local

# Directorios de caché
.npm
.eslintcache

# Logs
logs
*.log

# Sistema operativo
.DS_Store
Thumbs.db

Comparativa de métodos

Aspecto Express Generator Configuración Manual
Velocidad de inicio Rápido, estructura predefinida Más lento, requiere configuración
Estructura Estructura MVC básica predefinida Totalmente personalizable
Facilidad Simple, pocos comandos Más complejo, requiere más conocimientos
Aprendizaje Puede ocultar detalles de implementación Mejor para entender los fundamentos
Personalización Requiere modificar la estructura generada Estructura flexible desde el inicio
Ideal para Proyectos que siguen patrones estándar Proyectos con requisitos específicos

Scripts npm útiles

Estos son algunos scripts npm comunes que puedes añadir a tu package.json:

"scripts": {
  "start": "node app.js",
  "dev": "nodemon app.js",
  "debug": "nodemon --inspect app.js",
  "test": "jest",
  "lint": "eslint .",
  "format": "prettier --write \"**/*.js\""
}

Para usar estos scripts, necesitarás instalar las dependencias correspondientes:

usuario@linux:~/mi-proyecto-express$ npm install --save-dev nodemon jest eslint prettier

Buenas prácticas

  • Organización modular: Divide tu aplicación en módulos y componentes reutilizables.
  • Variables de entorno: Usa .env para configuraciones según el entorno (desarrollo, pruebas, producción).
  • Gestión de errores: Implementa middleware para capturar y manejar errores.
  • Validación: Valida siempre la entrada de usuario (puedes usar express-validator).
  • Pruebas: Escribe pruebas unitarias y de integración.
  • Documentación: Documenta tus API con herramientas como Swagger/OpenAPI.
  • Seguridad: Usa helmet y otras prácticas de seguridad recomendadas.

Solución de problemas comunes

Error: Puerto en uso

Error: listen EADDRINUSE: address already in use :::3000

Solución: Cambia el puerto o libera el puerto ocupado:

usuario@linux:~$ sudo lsof -i :3000
usuario@linux:~$ kill -9 PID
# O simplemente usa otro puerto en tu aplicación

Error: Módulo no encontrado

Error: Cannot find module 'express'

Solución: Instala las dependencias:

usuario@linux:~/mi-proyecto-express$ npm install

Cambios no se reflejan

Solución: Usa nodemon para reinicio automático:

usuario@linux:~/mi-proyecto-express$ npm install --save-dev nodemon
usuario@linux:~/mi-proyecto-express$ npm run dev
